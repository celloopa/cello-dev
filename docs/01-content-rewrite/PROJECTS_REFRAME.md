# Projects Reframe

## Goal

Reframe existing projects as product design proof rather than generic build work.

## Reframing Rules

Each project should answer:

- What user or workflow problem did this address?
- Who used or depended on it?
- What system, pattern, or prototype was designed?
- What constraints shaped the solution?
- What changed because of the work?

## Recommended Project Order

1. Figma/Payload Feature Prototype, once created
2. Ghosted
3. Ad Asset Directory
4. Ascendant Traders
5. VegAvengers
6. Visual Archive

## Current Project Framing

### Ghosted

Frame as an AI-assisted workflow product for technical job seekers, with keyboard-first UX, local-first constraints, JSON-based agent handoffs, and human-review boundaries.

### Ad Asset Directory

Frame as an internal product system for asset discovery, brand operations, non-technical admin workflows, and measurable lookup-time reduction.

### Ascendant Traders

Frame as a complex product-system case across acquisition, publishing, runtime state, diagnostics, risk communication, and reporting honesty.

### VegAvengers

Frame as mission-driven discovery with flexible search, mode switching, event-state handling, and responsive interaction design.

## Avoid

- Leading with stack names before product value
- Making every project sound like a coding demo
- Mixing visual archive work equally with product case studies
- Treating "built it myself" as the whole story
