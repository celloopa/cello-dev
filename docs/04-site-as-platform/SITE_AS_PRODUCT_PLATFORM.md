# Site As Product Platform

## Vision

The portfolio should become more than a static website.

It should become a product design platform where Cello can publish, organize, and connect:

- Case studies
- Technical prototypes
- Lab experiments
- Writing
- Videos
- Visual archive work
- GitHub projects
- Design systems
- Product thinking
- Work-in-progress updates

## Core Idea

The site should operate like a living product design operating system.

Each project can connect to:

- A case study
- A prototype
- A video walkthrough
- A GitHub repo
- A writing series
- Visual assets
- Lessons learned
- Future iterations

## Project Pipeline

Every major project can move through this pipeline:

```txt
Idea -> Research -> Prototype -> Lab Entry -> Case Study -> Video -> LinkedIn Posts -> Resume Proof
```

## Example: Figma/Payload Project

### Case Study

Full written product design story.

### Prototype

Payload-powered technical prototype.

### Video

Walkthrough explaining the product problem, design process, and demo.

### Writing

Process notes about design-dev handoff, data modeling, and prototyping.

### GitHub

Technical repo with product-focused README.

### LinkedIn

Build log, launch post, and reflections.

## Long-Term Goal

The site should help visitors understand not just what Cello made, but how Cello thinks, designs, communicates, prototypes, and improves product workflows.
