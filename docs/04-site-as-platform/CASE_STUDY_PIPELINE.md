# Case Study Pipeline

## Goal

Create a repeatable process for turning work into strong product design case studies.

## Pipeline

```txt
Idea -> Product Brief -> Prototype -> Evidence -> Case Study Draft -> Review -> Publish -> Promote
```

## Case Study Requirements

Each case study should include:

- Product problem
- Users or stakeholders
- Role
- Constraints
- Process
- Key decisions
- System or prototype details
- Outcomes
- Reflection

## Review Questions

- Is the product problem obvious?
- Is the user or workflow clear?
- Does the technical work support the design story?
- Are screenshots and captions doing useful proof work?
- Are outcomes honest and specific?
- Is there a strong reflection?
