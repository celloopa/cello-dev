# Content Review Checklist

- [ ] The target role is clear within 10 seconds.
- [ ] Product design appears before tool lists.
- [ ] Each project has a clear problem.
- [ ] Each project names users or stakeholders.
- [ ] Technical details explain design decisions.
- [ ] No unfinished work is presented as finished.
- [ ] No fake or unsupported metrics are used.
- [ ] CTAs point to the next useful action.
- [ ] Metadata and social preview copy are present.
- [ ] The page has no placeholder copy.
