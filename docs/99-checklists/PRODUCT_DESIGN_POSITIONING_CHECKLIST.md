# Product Design Positioning Checklist

- [ ] Homepage headline says Product Designer or product design clearly.
- [ ] The design-dev collaboration differentiator is visible.
- [ ] Case studies appear before visual archive work.
- [ ] The About section supports the same positioning.
- [ ] Resume summary matches the website story.
- [ ] Project cards emphasize product value.
- [ ] Technical fluency is framed as design leverage.
- [ ] WordPress/CMS experience is framed as content/product workflow experience.
- [ ] Visual craft supports the story without becoming the main identity.
