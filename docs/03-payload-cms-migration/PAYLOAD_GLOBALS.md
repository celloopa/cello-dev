# Payload Globals

## Goal

Define global CMS objects for site-wide settings and high-change editorial surfaces.

## Site Settings

Stores:

- Site name
- Site description
- Base URL
- Contact email
- Default Open Graph image
- Social links

## Navigation

Stores:

- Main navigation
- Footer navigation
- Primary CTA label and URL
- Optional featured link

## Homepage

Stores:

- Hero headline
- Hero subheadline
- Primary CTA
- Secondary CTA
- Featured case studies
- Featured lab projects
- Featured writing
- Featured visual archive items

## About Summary

Stores:

- Short bio
- Long bio
- Current focus
- Skills
- Tools
- Resume URL

## SEO Defaults

Stores:

- Default title template
- Default meta description
- Default OG image
- Twitter card defaults

## Rule

Use globals for editorial content that changes independently from page templates, not for every design token or component setting.
