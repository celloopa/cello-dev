# Admin Workflow

## Goal

Define the future editing workflow for a Payload-powered portfolio.

## Editorial Flow

1. Create draft.
2. Add required metadata.
3. Add media and alt text.
4. Connect related content.
5. Preview page.
6. Review against checklist.
7. Publish.
8. Share through LinkedIn, YouTube, GitHub, or Behance when appropriate.

## Admin Priorities

- Make case studies easy to draft and revise.
- Make related content obvious.
- Keep SEO and OG fields visible before publish.
- Support draft previews.
- Avoid overcomplicated fields that slow down writing.

## Roles

Initial implementation can assume one editor/admin.

Future roles may include:

- Admin
- Editor
- Reviewer

Do not overbuild permissions until there is a real need.
