# Drafts, Preview, And Versioning

## Goal

Define how the Payload-powered portfolio should handle drafts, previews, and content versions.

## Why This Matters

The site will include content that evolves over time:

- Case studies
- Writing
- Lab experiments
- Visual archive items
- Video pages
- Project pages

These should be editable as drafts before being published.

## Payload Features To Use

### Drafts

Use drafts for:

- Case studies in progress
- Articles in progress
- Lab entries not ready to publish
- Visual archive items being organized
- Video pages before launch

### Versions

Use versions for:

- Tracking changes to important case studies
- Reverting published content
- Reviewing edits over time

### Live Preview

Use preview for:

- Case study layouts
- Homepage updates
- Writing drafts
- Visual archive pages
- Lab/project pages

## Suggested Draft Workflow

1. Create content as draft.
2. Add required metadata.
3. Preview page.
4. Review copy and visuals.
5. Check accessibility and performance.
6. Publish.
7. Share through LinkedIn or YouTube.

## Content That Should Support Drafts

- Case Studies
- Pages
- Writing
- Lab Projects
- Visual Archive Items
- Videos
- Projects

## Publishing Checklist

Before publishing:

- [ ] Clear title
- [ ] Strong summary
- [ ] Correct status
- [ ] Featured image
- [ ] SEO title
- [ ] SEO description
- [ ] Open Graph image
- [ ] Proper tags
- [ ] Related content linked
- [ ] No placeholder text
- [ ] Accessibility checked
- [ ] Mobile layout checked
