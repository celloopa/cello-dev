## Body Structure

Use this order for current MDX case studies:

1. `CaseMeta` summary
2. Context
3. Problem
4. Users and stakeholders
5. Constraints
6. Product decisions
7. System or workflow model
8. Prototype or implementation details
9. Outcomes
10. Reflection

## Writing Rules

- Lead each section with product context.
- Use technical details to explain design decisions.
- Include media captions that describe what the artifact proves.
- Avoid unsupported metrics.
- End with what the work taught or what would be measured next.
